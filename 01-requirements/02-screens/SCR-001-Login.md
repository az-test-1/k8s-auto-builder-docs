## 3. 기능 정의

### 3.1 이메일/비밀번호 로그인

| ID | 기능 | 설명 |
|----|------|------|
| F001-01 | 이메일 입력 | 유효한 이메일 형식 검증 |
| F001-02 | 비밀번호 입력 | 최소 8자 이상 |
| F001-03 | 로그인 처리 | 인증 후 JWT 토큰 발급 |
| F001-04 | 로그인 유지 | 체크 시 Refresh Token 7일 유지 |
| F001-05 | 에러 표시 | 인증 실패 시 에러 메시지 표시 |

### 3.2 OAuth 로그인

| ID | 기능 | 설명 |
|----|------|------|
| F001-06 | GitHub OAuth | GitHub 계정으로 로그인 |
| F001-07 | Google OAuth | Google 계정으로 로그인 |
| F001-08 | 신규 사용자 | 최초 OAuth 로그인 시 자동 계정 생성 |

### 3.3 로그인 후 처리

| ID | 기능 | 설명 |
|----|------|------|
| F001-09 | 리다이렉트 | 로그인 전 요청 페이지로 이동 |
| F001-10 | 기본 이동 | 요청 페이지 없으면 대시보드로 이동 |

## 5. API 연동

### 5.1 로그인 API
```yaml
POST /api/v1/auth/login
Request:
  email: string
  password: string
  remember: boolean
Response:
  200:
    accessToken: string
    refreshToken: string
    user:
      id: string
      email: string
      name: string
      role: string
  401:
    message: "이메일 또는 비밀번호가 올바르지 않습니다"
```

### 5.2 OAuth API
```yaml
GET /api/v1/auth/oauth/{provider}
Params:
  provider: "github" | "google"
Response:
  302: Redirect to OAuth provider
```

## 6. 상태 관리

| 상태 | 타입 | 설명 |
|------|------|------|
| isLoading | boolean | 로그인 처리 중 |
| error | string | 에러 메시지 |
| redirectUrl | string | 로그인 후 이동할 URL |

---

## 7. 에러 처리

| 에러 코드 | 메시지 | 처리 |
|----------|--------|------|
| 401 | 인증 실패 | "이메일 또는 비밀번호가 올바르지 않습니다" 표시 |
| 403 | 계정 비활성화 | "비활성화된 계정입니다. 관리자에게 문의하세요" 표시 |
| 429 | 요청 초과 | "잠시 후 다시 시도해주세요" 표시 |
| 500 | 서버 오류 | "서버 오류가 발생했습니다" 표시 |

---

## 8. 접근성

- 탭 순서: 이메일 → 비밀번호 → 로그인 유지 → 로그인 버튼
- Enter 키로 폼 제출 가능
- 스크린 리더 지원 (aria-label)
